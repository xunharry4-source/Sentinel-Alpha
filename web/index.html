Behavior-Aligned Personal Trading Framework

Sentinel-Alpha

一个为个人交易者打造的多智能体系统。它不是追求最大收益的黑箱，而是把行为习惯、风险边界、实时市场和执行纪律统一到同一条决策链里。

8 Agents 3 Phases 4 Storage Layers
目标函数 U = E(R) - λσ² - φ(behavior)
系统重点 Behavior First, Execution Second

三阶段流程

  1. Simulation：压力场景与噪音干扰，测出用户真实反应。
  2. Synthesis：把行为弱点变成风险参数与仓位纪律。
  3. Execution：由执行代理落单，风控代理硬性兜底。

存储架构

按数据本质分层，而不是把所有状态塞进一个数据库。

存储类型 推荐技术 存储内容示例
关系型 (SQL) PostgreSQL 用户基本信息、实盘资产余额、订单状态
时序型 (TSDB) TimescaleDB 过去 10 年的 1 分钟线数据、用户模拟交易的行为轨迹
向量型 (Vector) Qdrant 用户心理侧写向量、金融新闻语义块、策略特征
缓存层 (Cache) Redis 实时盘口、Agent 通信队列、会话 Session

基础设施职责

PostgreSQL 保存可审计、可追责的交易状态。

TimescaleDB 保存高频行情和行为轨迹，支撑回放与分析。

Qdrant 保存用户记忆与语义检索上下文。

Redis 支撑低延迟缓存与多 Agent 协调。

编排层

LangChain 负责将检索、解释、摘要和 Agent 间上下文拼接成一条可维护的推理链。

它不替代交易逻辑，而是组织交易逻辑、记忆和解释输出。
